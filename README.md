#Explicação da lógica utilizada


O código desenvolvido tem como objetivo simular um sistema básico de monitoramento de uma missão espacial, utilizando a linguagem C. A lógica do programa foi construída para permitir que o usuário insira informações importantes da nave, visualize o estado atual da missão, execute análises automáticas dos dados e consulte um pequeno histórico de temperaturas registradas. O sistema funciona de forma interativa através de um menu principal, permitindo que as funcionalidades sejam acessadas repetidamente até que o usuário escolha encerrar a execução do programa.

No início do código são declaradas as variáveis principais responsáveis por armazenar os dados da missão. A variável “temperatura”, do tipo float, é utilizada para guardar valores decimais referentes à temperatura da nave. Já a variável “energia”, do tipo inteiro, armazena o percentual de energia disponível no sistema. A variável “comunicacao” também é do tipo inteiro e representa o estado da comunicação da nave, utilizando os valores 1 para indicar comunicação online e 0 para indicar comunicação offline. Além dessas variáveis, o programa também utiliza “opcao”, responsável por armazenar a escolha do usuário no menu principal, e “dadosInseridos”, que funciona como um controle lógico para verificar se já existem informações cadastradas no sistema antes de permitir análises ou visualizações.

Outro recurso importante utilizado é o vetor “historicoTemperaturas”, criado para armazenar até cinco registros de temperatura. O controle desse vetor é feito pela variável “contadorHistorico”, que indica quantas posições já foram preenchidas. Essa lógica permite que o sistema mantenha um pequeno histórico das temperaturas registradas ao longo da execução do programa.

A estrutura principal do sistema é baseada em um laço de repetição “do while”. Esse laço garante que o menu principal continue aparecendo continuamente até que o usuário escolha a opção de encerrar o sistema. A utilização do “do while” é adequada nesse caso porque o menu precisa ser executado pelo menos uma vez antes da condição de parada ser verificada. Dentro desse laço principal, o programa exibe todas as opções disponíveis ao usuário, incluindo inserir dados, visualizar status, executar análise, consultar histórico de temperaturas e finalizar o sistema.

Após o usuário escolher uma opção, o programa utiliza a estrutura “switch case” para decidir qual funcionalidade será executada. Essa estrutura torna o código mais organizado, permitindo separar claramente cada ação do sistema. Quando o usuário seleciona a opção de inserir dados da missão, o programa inicia uma sequência de validações para garantir que as informações digitadas sejam válidas.

A primeira validação é realizada na leitura da temperatura. Para isso, foi utilizado um laço infinito “while(1)” combinado com uma verificação do retorno da função “scanf”. O programa verifica se o valor digitado realmente corresponde a um número decimal válido. Caso o usuário digite letras ou caracteres inválidos, o sistema exibe uma mensagem de erro e limpa o buffer de entrada utilizando “getchar()”, evitando problemas na leitura seguinte. Essa lógica impede que valores incorretos sejam armazenados na variável temperatura.

Em seguida, o sistema solicita o nível de energia da nave. Nesse caso, foi utilizada uma estrutura “do while” para garantir que o valor digitado esteja entre 0 e 100. Se o usuário informar um número fora desse intervalo, o programa exibe uma mensagem informando que o valor é inválido e solicita uma nova entrada. Essa validação é importante porque o nível de energia representa uma porcentagem, portanto não pode assumir valores negativos ou superiores a 100.

A próxima etapa consiste na leitura do status da comunicação da nave. O sistema aceita apenas os valores 0 ou 1, representando respectivamente offline e online. Novamente é utilizada uma estrutura de repetição para obrigar o usuário a informar um valor válido. Essa lógica evita inconsistências no funcionamento do programa e mantém o controle correto do estado da comunicação.

Depois que todos os dados são inseridos corretamente, o programa salva a temperatura no vetor de histórico. Antes disso, ele verifica se ainda existe espaço disponível no vetor, utilizando a condição “contadorHistorico < 5”. Caso ainda haja espaço, a temperatura é armazenada na posição correspondente e o contador é incrementado. Em seguida, a variável “dadosInseridos” recebe o valor 1, indicando que já existem informações cadastradas no sistema e permitindo que as outras funcionalidades possam ser utilizadas.

Quando o usuário escolhe a opção de visualizar status, o sistema primeiro verifica se os dados já foram inseridos. Essa verificação é feita utilizando a variável de controle “dadosInseridos”. Caso ainda não existam dados cadastrados, o programa exibe uma mensagem informando isso ao usuário. Caso contrário, o sistema mostra na tela todas as informações atuais da missão, incluindo temperatura, nível de energia e estado da comunicação. Para deixar a exibição mais compreensível, o programa utiliza uma estrutura condicional “if else” para converter os valores numéricos da comunicação em mensagens textuais, mostrando “ONLINE” ou “OFFLINE”.

A funcionalidade de análise da missão é a parte principal da lógica do sistema. Nessa etapa, o programa avalia os dados inseridos e gera alertas automáticos com base nas condições estabelecidas. Primeiramente, a temperatura é analisada. Caso ela ultrapasse 80 graus, o sistema exibe uma mensagem de alerta indicando superaquecimento. Caso contrário, informa que a temperatura está dentro da faixa normal de funcionamento. Essa lógica simula um monitoramento real de segurança em sistemas espaciais.

Depois disso, o programa verifica o nível de energia da nave. Se a energia estiver abaixo de 20%, o sistema gera um alerta recomendando a ativação do modo de economia de energia. Caso contrário, informa que o nível energético está adequado. Essa análise demonstra como sistemas automatizados podem auxiliar na prevenção de falhas operacionais durante uma missão.

A comunicação também é monitorada. Caso o status esteja offline, o programa emite um alerta de falha de comunicação. Se a comunicação estiver online, o sistema informa que tudo está funcionando normalmente. Todas essas verificações utilizam estruturas condicionais “if else”, permitindo que o programa tome decisões com base nos dados fornecidos pelo usuário.

Outra funcionalidade importante é a exibição do histórico de temperaturas. Para isso, o sistema utiliza um laço “for”, responsável por percorrer todas as posições preenchidas do vetor de histórico. O contador define quantas leituras já foram armazenadas, evitando que posições vazias sejam exibidas. Assim, o programa apresenta ao usuário todas as temperaturas registradas durante a execução do sistema, funcionando como um pequeno registro de monitoramento da missão.

Por fim, quando o usuário escolhe a opção de encerrar o sistema, o programa exibe mensagens de finalização e o laço principal é encerrado. Isso acontece porque a condição do “do while” depende do valor da variável “opcao”. Quando a opção escolhida é 0, a condição “opcao != 0” torna-se falsa, encerrando a repetição e finalizando o programa.

De maneira geral, o código utiliza diversos conceitos fundamentais da programação em C, como variáveis, vetores, estruturas condicionais, estruturas de repetição, validação de entrada de dados e controle lógico de execução. A lógica implementada busca simular de forma simples o funcionamento de um sistema de monitoramento espacial, permitindo acompanhar informações importantes da missão, detectar situações de risco e auxiliar na tomada de decisões automáticas. Além disso, o programa demonstra boas práticas de programação, como validação de dados inseridos pelo usuário e organização das funcionalidades através do menu interativo.
